strcpy和memcpy

strcpy和mempcy都是标准C库函数，它们有下面的特点。
strcpy提供了字符串的复制。即strcpy只用于字符串复制，并且它不仅复制字符串内容之外，还会复制字符串的结束符。
memcpy提供了一般内存的复制。即memcpy对于需要复制的内容没有限制，因此用户更广。
strcpy的函数原型：
char strcpy(char dest, const char src);
memcpy的函数运行
char memcpy(void dest, const void src, size_t count);

strcpy和memcpy的相同点：
都是可以实现字符串拷贝的功能
strcpy和memcpy的不同点：
（1）实现功能不同。strcpy只能复制字符串，mempcy可以进行内存块间的拷贝，即可以拷贝存在内存块中的任意内容，例如字符数组、整型、结构体、类等；
（2）复制的方法不同。strcpy不需要指定长度，它遇到被复制字符的串结束符”\0”才结束，所以容易溢出。memcpy则是根据其第3个参数决定复制的长度。
（3）执行效率不同。memcpy最高，strcpy次之。

信号量，信号，互斥锁的区别

信号（signal）：是一种处理异步事件的方式。信号是比较复杂的通信方式，用于通知接收进程有某种事件发生，除了用于进程外，还可以发送信号给进程本身。Linux除了支持unix早期的信号语义函数，还支持语义函数符合posix.1标准的信号函数sigaction

信号量(semaphone)，进程间通信处理同步互斥的机制，是在多线程环境下使用的一种设施，它负责各个线程，以保证它们能够正确合理的使用公共资源。

互斥量和信号量的区别：
（1）互斥量的加锁和解锁必须由同一种线程完成，信号量可以由一线程释放，另一个线程得到。
（2）互斥量用于线程的互斥，信号量用于线程的同步。
（3）互斥量：是指某一资源同时允许一个访问者对其进行访问，具有唯一性和排他性。但互斥无法限制访问者对资源的访问顺序，即访问是无序的。
同步：是指在互斥的基础上（大多数基础），通过其它机制实现访问者对资源的有序访问。大多数情况下，同步已经实现互斥，特别是所有写入资源的情况下必须是互斥的。少数情况是指允许多个访问者同时访问资源。
（4）互斥量只能是0或1，信号量可以为非负整数。

Linux堆和栈的区别

  1. 栈区（stack），是由编译器自动分配释放，存放函数的参数值，局部变量的值等，局部变量的值等。其操作方式类似于数据结构中的栈。
  2. 堆区（heap），一般由程序员分配释放，若程序员不释放，程序结束时可能有OS回收。注意它与数据结构中的堆是两回事，分配方式类似于链表
  3. 因为栈是有编译器在需要的时候进行分配的，不需要时自动清除的变量存储区。

堆和栈的主要区别：

  • 管理方式：栈是自动管理的，堆是人为申请和释放
  • 空间大小不同
    （1）栈是向低地址扩展的数据结构，是连续的地址空间
    （2）堆是是向高地址扩展的数据结构，是不连续的地址空间
  • 是否产生碎片：
    （1）栈是不存在此问题的。
    （2）堆易出现此问题。
  • 增长方向：
    栈是从高到低的，堆是从低到高的。
  • 分配方式：
  • 分配效率不同

进程和线程的区别

  1. 进程是资源分配和调度的一个独立单元，而线程是cpu调度的基本单元。
  2. 同一个进程中可以包括多个线程，并且线程共享整个的资源（寄存器，堆栈，上下文），一个进程至少有一个线程
  3. 进程的创建调用fork或vfork，而线程的创建用pthread_create，进程结束后它拥有的所有线程都将销毁，而线程的技术不会影响同个进程中的其他线程的结束。
  4. 进程有自己的独立地址空间，每启动一个进程，系统就会为它分配地址空间，建立数据表来维护代码段，堆栈段和数据段，这样的操作非常昂贵。而线程是共享进程中的数据，使用相同的地址空间，因此CPU切换一个线程的花费比进程小很多。创建线程的开销比进程小。
  5. 进程间通信比线程间通信复杂。
  6. 多进程程序比多线程程序稳定，多进程因为各自有独立的地址空间，不会互相影响。
